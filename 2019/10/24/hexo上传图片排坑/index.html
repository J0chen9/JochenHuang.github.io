使用hexo-asset-image插件上传图片遇到的坑

在开心的搭建好hexo博客后，我满怀激动想写上自己今天的学习心得！当我想在博客中插入图片的时候，发现使用markdown写上本地路径是没办法上传到博客上的（小白程序员落下了泪水）。于是就在度娘上查阅资料，发现大佬们都建议使用的是hexo-asset-image插件的上传图片方式。

按着网上教程使用hexo-asset-image插件搭配设置_config.yml配置文件中的post_asset_folder: true。

然后把要引用的图片放在文章同名文件夹中（设置了post_asset_folder: true会自动建立一个与文章同名的文件夹，我们可以把该文章相关的所有资源放到这个文件夹中）。
最后在我们的markdown中只要使用![logo](文章同名文件夹/log.jpg)就可以插入图片

我发现我最终生成的html是<img src="/.com///1f4ebdc969ae126e9088b7c8dadd2f3.png">

在网上找了一天，终于给我找到了解决办法。是来自大佬的提供的解决方法。

这其实是hexo-asset-image插件的bug，要解决这个问题我们需要
一、先卸载插件（我是直接删除，发现是管用的）
二、重新下载 cnpm install https://github.com/xcodebuild/hexo-asset-image --save
三、打开文件 /node_modules/hexo-asset-image/index.js，修改js(一定要改)，把下面的代码复制进去即可
